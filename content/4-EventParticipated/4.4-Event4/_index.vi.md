## Tổng quan sự kiện

FCAJ Community Day lần này tập trung vào các chủ đề tiên tiến trong AI Agents, Voice AI và tự động hóa AWS DevOps.
Sự kiện quy tụ các nhà lãnh đạo ngành, founder và chuyên gia AWS chia sẻ kiến thức về xây dựng hệ thống AI agent,

Tìm hiểu sâu về mô hình microservices:
- **Khái niệm cơ bản Microservices**: Các khái niệm và nguyên tắc cốt lõi
- **Khi nào sử dụng Microservices**: Các trường hợp sử dụng phù hợp

#### Single Agent vs Multi-Agent Systems

**Khung quyết định quan trọng:**

**Khi nào dùng Single Agent:**
- Tác vụ đơn giản, tập trung
- Phạm vi hoạt động giới hạn
- Ra quyết định đơn giản
- Yêu cầu độ phức tạp thấp
- Cần chuyên môn một lĩnh vực

**Khi nào dùng Multi-Agent:**
- Vấn đề phức tạp, đa lĩnh vực
- Yêu cầu xử lý song song
- Các agent chuyên biệt cho từng tác vụ
- Nhu cầu mở rộng và chịu lỗi
- Tình huống ra quyết định hợp tác

**Cân nhắc chính:**
- **Độ phức tạp**: Độ phức tạp hệ thống vs agent
- **Khả năng mở rộng**: Nhu cầu tăng trưởng và mở rộng
- **Chi phí**: Chi phí phát triển và vận hành
- **Bảo trì**: Khả năng bảo trì dài hạn
- **Hiệu suất**: Yêu cầu về tốc độ và hiệu quả

### Diễn giả 9: Anh Toàn & Anh Nghi - Building Secure Private MCP cho Amazon Q

**Diễn giả:**
- **Anh Toàn**
- **Anh Nghi** (Diễn giả 2)

**Chủ đề:** Building Secure Private MCP (Model Context Protocol) cho Amazon Q

#### Giới thiệu về MCP (Model Context Protocol)

**MCP là gì?**
- Giao thức kết nối mô hình AI với nguồn dữ liệu bên ngoài
- Cách chuẩn hóa cung cấp ngữ cảnh cho mô hình AI
- Cho phép truy cập dữ liệu an toàn và được kiểm soát
- Hỗ trợ nhiều nguồn dữ liệu và định dạng

**Thành phần MCP:**
- **MCP Server**: Lưu trữ và quản lý context provider
- **MCP Client**: Kết nối đến server và truy xuất ngữ cảnh
- **Context Provider**: Adapter cho nguồn dữ liệu khác nhau
- **Lớp bảo mật**: Xác thực và phân quyền

**Tại sao MCP quan trọng:**
- Đảm bảo bảo mật và quyền riêng tư dữ liệu
- Cung cấp truy cập được kiểm soát vào thông tin nhạy cảm
- Cho phép ngữ cảnh nhất quán giữa các công cụ AI khác nhau
- Hỗ trợ yêu cầu tuân thủ


#### Giải pháp VPC Connection cho Amazon Q

**Thách thức bảo mật:**
Làm thế nào để kết nối Amazon Q với nguồn dữ liệu riêng tư một cách an toàn mà không để lộ thông tin nhạy cảm ra internet công cộng?

**Kiến trúc VPC Connection:**

**Thành phần:**
1. **Amazon Q trong AWS VPC**: Triển khai Q trong mạng riêng
2. **VPC Endpoint**: Kết nối an toàn đến dịch vụ AWS
3. **MCP Server trong VPC**: Lưu trữ MCP server trong subnet riêng
4. **Nguồn dữ liệu**: Database nội bộ, file system, API

**Lợi ích bảo mật:**
- **Không tiếp xúc công cộng**: Dữ liệu không bao giờ rời khỏi mạng riêng
- **Mã hóa truyền tải**: Tất cả truyền thông được mã hóa
- **Kiểm soát truy cập**: Phân quyền IAM chi tiết
- **Audit Logging**: Dấu vết audit đầy đủ
- **Tuân thủ**: Đáp ứng yêu cầu quy định

**Các bước triển khai:**
1. Thiết lập VPC với subnet phù hợp
2. Triển khai MCP server trong subnet riêng
3. Cấu hình VPC endpoint cho dịch vụ AWS
4. Thiết lập security group và NACL
5. Kết nối Amazon Q với MCP server
6. Cấu hình tích hợp nguồn dữ liệu
7. Triển khai xác thực và phân quyền
8. Bật logging và monitoring

#### Video Demo: MCP Server hoạt động

**Điểm nổi bật Demo:**

**Thiết lập và cấu hình:**

cơ sở hạ tầng cloud.

một cách an toàn mà không ảnh hưởng bảo mật.
